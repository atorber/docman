数据集提供数据导入能力，帮助用户将数据导入到已有数据集中，实现数据集、权重等文件的上传到百舸进行训练、开发等业务操作。

目前数据集支持已有数据集导入和自定义下载导入：

|导入方式|概述|数据集存储类型支持|
|-|-|-|
|已有数据集导入|数据集管理提供存储类型为对象存储BOS和并行文件存储PFS（L2）数据集之间相互导入数据，通过已有数据导入功能可以方便地将数据在PFS和BOS之间进行转存。|PFS（极速型L2）、BOS|
|自定义下载|通过编写自定义下载任务脚本，从任意百舸资源池网络可达的数据源下载或转储数据到数据集。|数据集支持的全部存储类型|

进入数据集详情页，点击"版本列表"Tab，在版本列表中操作"导入数据"。

## 已有数据集导入

![自定义数据集-数据导入1.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/%E8%87%AA%E5%AE%9A%E4%B9%89%E6%95%B0%E6%8D%AE%E9%9B%86-%E6%95%B0%E6%8D%AE%E5%AF%BC%E5%85%A51_a29df36.png)

### 限制条件

当前仅支持PFS L2与BOS之间进行互相转储。

### 操作步骤

1. 将需要导入数据的目录（目标数据集）和被导入目录（源数据集）分别创建为数据集，数据集创建完成后在数据集列表中找到对应数据集，在操作列选择"导入数据"
2. 在创建转储任务弹窗内选择源数据集及版本等信息。

> 例如：将数据集B（BOS类型）的v1版本（存储路径/dataset/test）数据导入到数据集A（PFS类型）的v1版本（存储路径/dataset/test/v1），则在数据集A操作"导入数据"

|参数|说明|
|-|-|
|任务名称|转储任务的名称，支持字母、数字、下划线_、中划线-，需以字母开头，长度为1~128个字符|
|源数据集|要导入的数据集，可选择当前用户有读权限的数据集|
|源数据版本|源数据集的数据版本，每个版本对应一个存储路径|
|冲突策略|可选择以下策略：<br>- 覆盖该文件：用源数据集的文件覆盖目标数据集中的同名文件<br>- 跳过该文件：不导入该文件<br>- 保留两者：在目标数据集中同时保存两个同名数据文件|
|目标数据集|当前操作的数据集|
|目标数据版本|导入数据存储的版本，每个版本对应一个存储路径|

3. 提交转储任务，在数据集详情的"转储任务"Tab页可以查看转储任务记录及状态。

## 自定义下载导入

自定义下载是利用用户自有资源池以任务的形式下载网络上的数据到当前数据集。

1. 通过编写合适的启动命令，可以实现从任意远程数据源下载数据；
2. 任务使用资源池的CPU和内存资源，不需要使用GPU资源。

![自定义数据集-数据导入2.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/%E8%87%AA%E5%AE%9A%E4%B9%89%E6%95%B0%E6%8D%AE%E9%9B%86-%E6%95%B0%E6%8D%AE%E5%AF%BC%E5%85%A52_b0c817d.png)

### 限制条件

需要资源池可以访问需要下载的远程数据源，下载HuggingFace、GitHub等国外网站资源需先自行设置资源池可以访问外网或使用国内源。

### 操作步骤

1. 填写下载任务表单信息

|参数|说明|
|-|-|
|镜像地址|下载数据需要使用的镜像，支持百舸预置镜像和自定义镜像。一般情况下如无特殊需求，选择任意百舸预置镜像均可以完成下载|
|下载脚本|下载任务的执行脚本，脚本中包含下载数据的全部逻辑，包括鉴权、断点续传等<br>特别注意：<br>任务会将当前数据集版本对应的路径默认挂载到/mnt/dataset，下载脚本中需要使用该路径及其子路径保存下载的数据，同时默认注入环境变量DATASET_PATH=/mnt/dataset，也可以通过引用DATASET_PATH使用默认挂载路径|
|资源池类型|运行下载任务的资源池类型|
|资源队列|运行下载任务的资源队列|
|资源配额|运行任务的资源限制，默认值4核8G，可以根据下载脚本设置的并发数进行调整，如果不设置则不会限制|

2. 提交下载任务成功后，在"分布式训练"任务列表中可以查看任务的日志和事件观察进度和异常。

> ⚠️ **安全提示**：自定义下载脚本会以任务形式执行，请确保：
> 1. 下载脚本来源可信，避免执行恶意代码
> 2. 脚本中不要包含硬编码的密码、密钥等敏感信息
> 3. 建议先在本地测试脚本，验证后再在生产环境使用

### 下载脚本示例

> 特别注意：
> 1. 以下示例均在网络可达的前提条件下验证，使用前请先验证资源池可以访问源数据地址
> 2. 验证使用镜像registry.baidubce.com/aihcp-public/pytorch:2.1.0-cu12-py3.10-ubuntu22.04，自定义其他镜像时可根据事件信息进行脚本调试
> 3. 示例脚本只是方便用户理解下载方法，实际使用时需要根据需求对脚本进行修改

#### 从魔搭下载数据集、权重

```bash
#!/bin/bash

# 设置数据集名称
DATASET_NAME="liucong/Chinese-DeepSeek-R1-Distill-data-110k"

# 构建目标存储路径，以数据集根目录/数据集名称存储
TARGET_DIR="$DATASET_PATH/$DATASET_NAME"

# 检查是否已安装 modelscope
if ! command -v modelscope >/dev/null 2>&1; then
    echo "modelscope 未安装，正在安装..."
    # 使用 --user 避免权限问题（可选）
    pip install --user modelscope
    # 将 --user 安装的 bin 加入 PATH（适用于某些环境）
    export PATH="$HOME/.local/bin:$PATH"
fi

# 使用 modelscope CLI 下载数据集（注意：使用 --dataset 而非 --model）
echo "正在下载数据集 $DATASET_NAME 到 $TARGET_DIR ..."
modelscope download --dataset "$DATASET_NAME" --revision master --local_dir "$TARGET_DIR"

if [ $? -eq 0 ]; then
    echo "数据集已成功下载至: $TARGET_DIR"
else
    echo "下载失败！请检查网络、权限或数据集是否存在。"
    exit 1
fi
```

#### 从HuggingFace下载数据集

```bash
#!/bin/bash

# 配置
HF_DATASET_ID="Congliu/Chinese-DeepSeek-R1-Distill-data-110k"

# 构建本地目标路径
LOCAL_DIR="$DATASET_PATH/$HF_DATASET_ID"
mkdir -p "$LOCAL_DIR"

# 检查是否已存在（简单判断）
if [ -f "$LOCAL_DIR/.hf_download_complete" ]; then
    echo "数据集已下载（检测到 .hf_download_complete 标记），跳过。"
    exit 0
fi

# 安装 huggingface_hub（如果未安装）
if ! python3 -c "import huggingface_hub" &> /dev/null; then
    echo "正在安装 huggingface_hub..."
    pip install --quiet huggingface_hub
fi

# 创建临时 Python 脚本
PY_SCRIPT=$(cat <<EOF
from huggingface_hub import snapshot_download
import os

local_dir = os.environ['LOCAL_DIR']
repo_id = os.environ['HF_DATASET_ID']

print(f"正在从 Hugging Face 下载数据集: {repo_id}")
snapshot_download(
    repo_id=repo_id,
    local_dir=local_dir,
    repo_type="dataset",
    max_workers=8
)
print("下载完成！")
EOF
)

# 导出环境变量供 Python 使用
export LOCAL_DIR="$LOCAL_DIR"
export HF_DATASET_ID="$HF_DATASET_ID"

# 执行下载
echo "开始从 Hugging Face Hub 下载数据集..."
python3 -c "$PY_SCRIPT"

# 检查是否成功并创建标记文件
if [ $? -eq 0 ]; then
    touch "$LOCAL_DIR/.hf_download_complete"
    echo "数据集已保存至: $LOCAL_DIR"
else
    echo "下载失败。请检查数据集 ID、网络或访问权限。"
    exit 1
fi
```

#### 从开放网址下载

```bash
#!/bin/bash

# 配置：通过环境变量传入
HTTP_URL="https://example.com/data/SKYLENAGE-ReasoningMATH.zip"
DATASET_NAME="SKYLENAGE-ReasoningMATH"

# 构建本地目标目录
LOCAL_DIR="$DATASET_PATH/Alibaba-DT/$DATASET_NAME"
mkdir -p "$LOCAL_DIR"

# 推断文件名（从 URL 最后一段）
FILENAME=$(basename "$HTTP_URL")
TMP_FILE="$LOCAL_DIR/${FILENAME}"

# 检查是否已下载
if [ -f "$TMP_FILE" ]; then
    echo "文件 $TMP_FILE 已存在，跳过下载。"
    exit 0
fi

# 下载文件
echo "正在从 $HTTP_URL 下载..."
if command -v wget &> /dev/null; then
    wget -O "$TMP_FILE" --progress=bar:force:noscroll "$HTTP_URL"
elif command -v curl &> /dev/null; then
    curl -L -o "$TMP_FILE" -# "$HTTP_URL"
else
    echo "错误: 未找到 wget 或 curl，请安装其中一个工具。"
    exit 1
fi

# 检查下载是否成功
if [ $? -ne 0 ] || [ ! -s "$TMP_FILE" ]; then
    echo "下载失败或文件为空。"
    rm -f "$TMP_FILE"
    exit 1
fi

# 自动解压（如果为压缩包）
if [[ "$FILENAME" == *.zip ]]; then
    echo "正在解压 ZIP 文件..."
    unzip -q -o "$TMP_FILE" -d "$LOCAL_DIR"
elif [[ "$FILENAME" == *.tar.gz ]] || [[ "$FILENAME" == *.tgz ]]; then
    echo "正在解压 TAR.GZ 文件..."
    tar -xzf "$TMP_FILE" -C "$LOCAL_DIR"
elif [[ "$FILENAME" == *.tar.bz2 ]]; then
    echo "正在解压 TAR.BZ2 文件..."
    tar -xjf "$TMP_FILE" -C "$LOCAL_DIR"
else
    echo "文件非压缩格式，已保存至: $TMP_FILE"
fi

echo "数据集已准备完毕，路径: $LOCAL_DIR"
```

## 常见问题

### 1. 导入任务失败怎么办？
- 检查源数据集是否有读权限
- 确认存储类型是否支持相互转储（目前仅支持PFS L2与BOS）
- 查看转储任务日志，获取具体错误信息

### 2. 自定义下载任务进度如何查看？
- 在"分布式训练"任务列表中查看任务状态
- 点击任务查看详细日志和事件

### 3. 资源池无法访问外网怎么办？
- 使用国内镜像源（如ModelScope国内源）
- 配置代理设置
- 先下载到本地，再从本地上传

### 4. 下载脚本执行时间过长？
- 检查资源配额设置，可以适当增加资源
- 优化下载脚本，使用多线程或断点续传
- 检查网络连接是否稳定

### 5. 数据导入后如何验证？
- 在数据集详情页查看数据版本列表
- 使用文件浏览功能查看导入的文件
- 在训练任务中挂载数据集进行测试
