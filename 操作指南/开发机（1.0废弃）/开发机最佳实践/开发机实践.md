## 使用conda+PFS开发及提交任务
在AI模型开发场景中，可以使用conda 统一管理 AI 作业的环境依赖。conda安装在PFS中，保证开发机和训练集群共享envs，所有任务均使用一个基础镜像在容器内按需选择conda env，而无需为每个开发项目打包镜像。

![conda安装在PFS中.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/conda%E5%AE%89%E8%A3%85%E5%9C%A8PFS%E4%B8%AD_d6f5dcb.png)

**以下以一个简单的 DeeepSpeed nccl-test 任务为例，说明如何将开发完成的代码在集群中提交为任务。**
### 在开发机中创建目标环境

1. 打开开发机，进入Terminal

![打开开发机命令行.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/%E6%89%93%E5%BC%80%E5%BC%80%E5%8F%91%E6%9C%BA%E5%91%BD%E4%BB%A4%E8%A1%8C_aa6f3d9.png)

2. 将 conda 安装到 PFS 共享存储中，默认部署开发机 PFS 挂载在`/mnt/cluster`，Conda 安装在 `/mnt/cluster/test/conda`中

```bash
# 下载 miniconda
wget http://mirrors.baidubce.com/anaconda/archive/Anaconda3-2023.03-Linux-x86_64.sh

#安装 miniconda 到 PFS 目录，默认pfs挂载路径为 /mnt/cluster为共享文件
bash Anaconda3-2023.03-Linux-x86_64.sh -p /mnt/cluster/test/conda
# -p 参数指定为 conda 安装的目录 prefix，需指定一个 PFS 的路径
# 最后一步会提示输入 yes 在 .bashrc 中默认初始化 conda 环境

# 初始化conda
source /mnt/cluster/test/conda/etc/profile.d/conda.sh
```

> 如存在网络问题需要修改conda源为国内源，具体方法可自行网络搜索

3. 创建PyTorch + DeepSpeed的conda环境


```bash
# 创建名为 ds 的 conda 环境，指定python版本
conda create --name ds python=3.8
# 激活此 conda 环境
conda activate ds
# 为该环境安装 pip，后续则可使用 pip 在次环境安装依赖
conda install pip

# 安装前检查 pip 源是否为百度源
pip config list
# 如果不是百度源执行下面命令
pip config set global.index-url http://mirrors.baidubce.com/pypi/simple
pip config set global.extra-index-url http://mirrors.baidubce.com/pypi/simple

# 安装 torch + deepspeed
pip install torch -i http://mirrors.baidubce.com/pypi/simple/ --trusted-host mirrors.baidubce.com
pip install deepspeed -i http://mirrors.baidubce.com/pypi/simple/ --trusted-host mirrors.baidubce.com

# 在激活的 conda 环境中安装 ipykernel，它允许你将这个环境添加为 Jupyter Notebook 的内核
conda install ipykernel
# 将 Conda 环境添加为 Jupyter 内核
python -m ipykernel install --user --name ds --display-name "Python (ds)"
# --name myenv 是内核的名称。
#--display-name "Python (ds)" 是在 Jupyter Notebook 中显示的名称。
```

### 使用YAML方式提交多机任务
以llama2-7b模型训练为例，权重和数据集存放在/mnt/cluster目录下，实际使用时根据需要修改为自己的路径地址

1. 在 PFS 中创建一个 CCE 环境的多机任务启动脚本（本例将运行脚本保存与 PFS 的 `/mnt/cluster/test/run.sh`中），该脚本在base镜像的容器中使用 PFS 上保存的与开发机相同的conda ds环境
```bash
#!/bin/bash

# -------------------------------
# 在原有的启动脚本中增加以下加载conda环境的命令，其他命令保持不变
# 初始化 PFS 中的 conda
source /mnt/cluster/test/conda/etc/profile.d/conda.sh
# 激活与开发机上相同conda 环境ds
conda activate ds
# -------------------------------

cd /mnt/cluster/test/code/DeepSpeedExamples/benchmarks/communication

torchrun --nnodes $WORLD_SIZE --node-rank $RANK --nproc-per-node 8 --master-addr $MASTER_ADDR --master-port $MASTER_PORT all_reduce.py --warmups 5 --trials 200 --dist torch --maxsize=28 --debug --scan
```

2. 提交测试任务

修改yaml文件启动命令 command 修改为第一步中保存的启动脚本
```yaml
apiVersion: "kubeflow.org/v1"
kind: PyTorchJob
metadata:
  name: ds-nccl-test # 任务名称，提交不同任务时需修改不同的任务名
  namespace: default
spec:
  pytorchReplicaSpecs:
    Master:
      replicas: 1 # 分布式任务中 Master 数量固定 1，就是 RANK=0 的特殊 Worker 也参与计算
      restartPolicy: OnFailure
      template:
        spec:
          containers:
            - name: pytorch
              image: registry.baidubce.com/cce-ai-native/pytorch:22.08-py3-ds-example
              resources:
                limits:
                  nvidia.com/gpu: 8
                  rdma/hca: "1"
              env:
                - name: NCCL_DEBUG
                  value: INFO
              command:
              - /bin/bash
              - /mnt/cluster/test/run.sh # 启动脚本位于 PFS
              volumeMounts:
                - mountPath: /dev/shm
                  name: shm
                - mountPath: /mnt/cluster # PFS 存储在容器中的挂载点，与开发机挂载保持一致
                  name: data
              securityContext:
                capabilities:
                  add: ["IPC_LOCK"]
          schedulerName: volcano
          volumes:
            - hostPath:
                path: /dev/shm
              name: shm
            - name: data
              persistentVolumeClaim:
                claimName: pvc-pfs
    Worker:
      replicas: 3 # 此处可修改为期望的 Worker 数量，此例为 Master+Worker (1+3)*8 共 32 张卡
      restartPolicy: OnFailure
      template:
        spec:
          containers:
            - name: pytorch
              image: registry.baidubce.com/cce-ai-native/pytorch:22.08-py3-ds-example
              env:
                - name: NCCL_DEBUG
                  value: INFO
              resources:
                limits:
                  nvidia.com/gpu: "8"
                  rdma/hca: "1"
              command:
              - /bin/bash
              - /mnt/cluster/test/run.sh # 启动脚本位于 PFS
              volumeMounts:
                - mountPath: /dev/shm
                  name: shm
                - mountPath: /mnt/cluster # PFS 存储在容器中的挂载点，与开发机挂载保持一致
                  name: data
              securityContext:
                capabilities:
                  add: ["IPC_LOCK"]
          schedulerName: volcano
          volumes:
            - hostPath:
                path: /dev/shm
              name: shm
            - name: data
              persistentVolumeClaim:
                claimName: pvc-pfs
```
运行`kubectl apply -f ./job.yaml`提交任务到集群

## 构建自定义基础镜像
使用conda on PFS提交任务最佳实践中方式进行训练时我们需要选择一个环境，并将这个基础环境制作成一个基础镜像供后续训练使用。
以下介绍如何基于百舸基础镜像创建自定义镜像，并利用自定义镜像创建百舸训练任务。
### 前置准备

- 开通百度容器镜像服务-CCR（[https://cloud.baidu.com/product/ccr.html](https://cloud.baidu.com/product/ccr.html)）（下图中 test-bj为命名空间名称，pytorch-deepspeed为镜像仓库名称）

![开通ccr1.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/%E5%BC%80%E9%80%9Accr1_bb20a77.png)

- 在一台有访问外网权限且装有docker环境的办公PC或本地开发机（非所述的Jupyter Notebook开发机）上登录百度容器镜像服务

![开通ccr2.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/%E5%BC%80%E9%80%9Accr2_bcfa662.png)

```bash
# 登录百度容器镜像服务
# 示例 docker login --username=<百度智能云用户名> <实例名称>-<公网或私有网络访问域名>
docker login --username=xxxxxx ccr-2p9lyswe-pub.cnc.bj.baidubce.com
```
如果有可以远程访问的百度云主机节点，也可以使用云主机进行操作
### 自定义镜像制作
登录百舸控制台，在 AI加速套件 菜单下，找到要使用的镜像，获取镜像地址

![自定义制作镜像1.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/%E8%87%AA%E5%AE%9A%E4%B9%89%E5%88%B6%E4%BD%9C%E9%95%9C%E5%83%8F1_51d74ae.png)

现在以v2.0.1镜像为基础，制作支持自定义镜像（命令中的容器名称、镜像名称、仓库地址需要替换为用户实际名称）
```bash
#启动docker容器
docker run  -itd --net=host --ipc=host --name llm_test --runtime=nvidia registry.baidubce.com/aihc-aiak/aiak-training-llm:ubuntu22.04-cu12.3-torch2.2.0-py310-bccl1.2.4.1_v2.0.1.0_release /bin/bash
#进入docker容器
docker exec -it llm_test bash

#安装deepspeed库（使用百度镜像源加速）
pip3 install deepspeed -i http://mirrors.baidubce.com/pypi/simple/ --trusted-host mirrors.baidubce.com
#下载测试代码，放在/workspace下
git clone https://github.com/microsoft/DeepSpeedExamples.git
#退出容器
exit

#将当前容器打包成镜像存到本地仓库
docker commit llm_test pytorch:v1.0
#查看本地镜像列表，会有名为pytorch的镜像
docker images 
#标记本地pytorch镜像，镜像id：0b8cffe6e027
docker tag 0b8cffe6e027 ccr-2p9lyswe-pub.cnc.bj.baidubce.com/test-bj/pytorch:v1.0
#将标记后镜像推送至实例镜像仓库
docker push ccr-2p9lyswe-pub.cnc.bj.baidubce.com/test-bj/pytorch:v1.0
```
推送后，CCR仓库中会出现我们自定义的镜像

## 数据/文件上传到开发机
开发机与训练集群使用相同的PFS存储保存数据，因此开发过程中需要使用的权重、数据集、代码等文件只需要上传到PFS中即可在开发机中操作使用
百度云PFS产品提供从百度云对象存储BOS中转存文件，建议将需要使用的文件先上传到对象存储再转存储到PFS

**特别注意：开发机从非当前账号下的对象存储中下载文件须具备访问外网的权限**

1. 安装BOS命令行工具
```bash
# 下载bcecmd程序
wget https://doc.bce.baidu.com/bce-documentation/BOS/linux-bcecmd-0.3.8.zip
# 解压
unzip linux-bcecmd-0.3.8.zip
cd linux-bcecmd-0.3.8
```
2. 下载数据到PFS
在bcecmd中执行如下操作完成将百舸demo数据（存储在百度云对象存储BOS）同步到PFS，存放在/mnt/cluster目录下（以下以LLaMA 7B训练模型及数据下载为例）：
```bash
# 数据及模型下载用以下命令
# 下载测试数据集
./bcecmd --conf-path conf bos sync bos://cce-ai-datasets/cce-ai-datasets.bj.bcebos.com/megatron_llama/pile_llama_test/ /mnt/cluster/llama/pile_llama_test/
# 下载模型参数权重
./bcecmd --conf-path conf bos sync bos://cce-ai-datasets/cce-ai-datasets.bj.bcebos.com/megatron_llama/megatron_llama_7b_checkpoint_tp1_pp1_dp8_zero1/ /mnt/cluster/llama/megatron_llama_7b_checkpoint_tp1_pp1_dp8_zero1/
# 下载tokenizer
./bcecmd --conf-path conf bos sync bos://cce-ai-datasets/cce-ai-datasets.bj.bcebos.com/megatron_llama/llama_tokenizer/ /mnt/cluster/llama/tokenizer/
```
更多操作方法参考BOS、PFS产品操作手册
