此文档讲解如何使用分布式训练任务开发自己的业务任务模板。

在进行工作流编排前，开发者需要先开发单个任务，将工作流中的每个单个任务在分布式训练任务中进行提交验证，一旦单个任务顺利跑通，继续将任务制作为任务模板即可在工作流中被复用，沉淀为AI资产。

## 任务模板开发

工作流的CustomTask对应百舸的分布式训练任务，所以百舸CustomTask类型的任务模板支持的参数与分布式训练任务完全一致。

### 在分布式训练任务中跑通任务

1. 任务代码开发完成后，进入百舸控制台，点击左侧【分布式训练】，点击页面的【创建任务】填写表单验证任务。创建训练任务详细步骤参考 [创建训练任务](https://cloud.baidu.com/doc/AIHC/s/Tlib5k9l1)


![](https://rte.weiyun.baidu.com/wiki/attach/image/api/imageDownloadAddress?attachId=02cf097c5ee34080a0ae6f2234bb3259&docGuid=I9KxBlpES3rGNL)
2. 任务跑通后，将任务参数信息整理为工作流任务基础模板，即将创建任务时填写的表单项使用接口参数方式表达，任务参数可以在任务详情页查看。

> 1. 以下模板主要以数据处理场景进行演示
> 2. 完整的任务参数模板可参考 任务模板示例/固定参数任务模板

### 设计任务模板参数
将任务中可以外部传入的参数设置为input参数，推荐的设计原则：

1. 如果不同任务使用的资源量不同，将资源参数提取为input
2. 所有的任务逻辑控制参数，放置在envs中传入

上述示例中我们将环境变量中定义的数据目录（SOURCE_DATA_DIR、TARGET_DATA_DIR）、执行命令（command）、BOS挂载地址设置为模板的可传入参数


```yaml
version: v1
kind: PipelineTemplate
taskTemplates:
  - name: custom-task-demo
    type: CustomTask
    inputs:
      - name: command # 参数名
        type: string  # 参数类
        hint: 执行命令  # 参数描述
      - name: source_data_dir # 参数名
        type: string # 参数类型
        hint: 源数据目录 # 参数描述
      - name: target_data_dir # 参数名
        type: string # 参数类型
        hint: 目标数据目录 # 参数描述
      - name: bos_dir # 参数名
        type: string # 参数类型
        hint: BOS存储地址 # 参数描述
    spec:
      queue: aihcq-h1plvpzb5gh0
      jobType: PyTorchJob
      command: '{{inputs.parameters.command}}'
      jobSpec:
        replicas: 1
        image: registry.baidubce.com/aihc-aiak/aiak-megatron:ubuntu20.04-cu11.8-torch1.14.0-py38_v1.2.7.12_release
        resources: []
        envs:
          - name: SOURCE_DATA_DIR
            value: '{{inputs.parameters.source_data_dir}}'
          - name: TARGET_DATA_DIR
            value: '{{inputs.parameters.target_data_dir}}'
      labels: []
      datasources: 
        - type: bos
          name: ""
          sourcePath: '{{inputs.parameters.bos_dir}}'
          mountPath: /mnt/bos/data
tasks:
  - name: task_a
    taskTemplateName: custom-task-demo
    inputs:
      - name: command
        value: echo "This is a defined command."
      - name: source_data_dir
        value: bos://my-bucket/datasets/source
      - name: target_data_dir
        value: bos://my-bucket/datasets/target
      - name: bos_dir
        value: bos://my-bucket/datasets
```

### 数据集下载

#### 从魔搭下载数据集

参数说明：

|参数名称|说明|
|-|-|
|queue_id|队列ID，示例：aihcq-h1plvp|    
|dataset_name|魔搭社区的数据集名称，示例：liucong/Chinese-DeepSeek-R1-Distill-data-110k|
|target_data_dir|数据集的保存目录，示例：bos://my-bucket/datasets|

yaml模板：

```yaml
version: v1
kind: PipelineTemplate

taskTemplates:
  - name: template-dataset-download-modelscope
    type: CustomTask
    inputs:
      - name: queue_id
        type: string
        hint: 队列ID
      - name: dataset_name
        type: string
        hint: 数据集名称
      - name: target_data_dir
        type: string
        hint: 数据集在BOS的存储目录
    spec:
      queue: '{{inputs.parameters.queue_id}}'
      jobType: PyTorchJob
      command: |
        #!/bin/sh

        # 检查是否已安装 modelscope
        if ! command -v modelscope >/dev/null 2>&1; then
            echo "modelscope 未安装，正在安装..."
            pip install --user modelscope
            export PATH="$HOME/.local/bin:$PATH"
        fi

        # 使用 modelscope CLI 下载数据集
        echo "正在下载数据集 $DATASET_NAME..."
        modelscope download --dataset "$DATASET_NAME" --revision master --local_dir /mnt/bos/data

        if [ $? -eq 0 ]; then
            echo "数据集已成功下载至: /mnt/bos/data"
        else
            echo "下载失败！请检查网络、权限或数据集是否存在。"
            exit 1
        fi
      jobSpec:
        image: registry.baidubce.com/aihcp-public/pytorch:2.7.0-cu12.8.61-py3.12-ubuntu24.04
        replicas: 1
        envs:
          - name: DATASET_NAME
            value: '{{inputs.parameters.dataset_name}}'
      datasources:
        - type: bos
          name: ""
          sourcePath: '{{inputs.parameters.target_data_dir}}'
          mountPath: /mnt/bos/data

tasks:
  - name: job-1
    taskTemplateName: template-dataset-download-modelscope
    inputs:
      - name: queue_id
        value: aihcq-h1plvp
      - name: dataset_name
        value: liucong/Chinese-DeepSeek-R1-Distill-data-110k
      - name: target_data_dir
        value: bos://my-bucket/datasets
```

#### 从HuggingFace下载数据集

从HuggingFace下载数据集，需要自行保证集群可以正常访问HuggingFace

参数说明：

|参数名称|说明|
|-|-|
|queue_id|队列ID，示例：aihcq-h1plvp|    
|dataset_name|魔搭社区的数据集名称，示例：liucong/Chinese-DeepSeek-R1-Distill-data-110k|
|target_data_dir|数据集的保存目录，示例：bos://my-bucket/datasets|

yaml模板：

```yaml
version: v1
kind: PipelineTemplate

taskTemplates:
  - name: template-dataset-download-huggingface
    type: CustomTask
    inputs:
      - name: queue_id
        type: string
        hint: 队列ID
      - name: dataset_name
        type: string
        hint: 数据集名称
      - name: target_data_dir
        type: string
        hint: 数据集在BOS的存储目录
    spec:
      queue: '{{inputs.parameters.queue_id}}'
      jobType: PyTorchJob
      command: |
        #!/bin/sh

        # 构建本地目标路径
        LOCAL_DIR="/mnt/bos/data/$DATASET_NAME"
        mkdir -p "$LOCAL_DIR"

        # 检查是否已存在（简单判断）
        if [ -f "$LOCAL_DIR/.hf_download_complete" ]; then
            echo "数据集似乎已下载（检测到 .hf_download_complete 标记），跳过。"
            exit 0
        fi

        # 安装 huggingface_hub（如果未安装）
        if ! python3 -c "import huggingface_hub" &> /dev/null; then
            echo "正在安装 huggingface_hub..."
            pip install --quiet huggingface_hub
        fi

        # 创建临时 Python 脚本
        PY_SCRIPT=$(cat <<EOF
        from huggingface_hub import snapshot_download
        import os

        local_dir = os.environ['LOCAL_DIR']
        repo_id = os.environ['DATASET_NAME']

        print(f"正在从 Hugging Face 下载数据集: {repo_id}")
        snapshot_download(
            repo_id=repo_id,
            local_dir=local_dir,
            repo_type="dataset",
            max_workers=8
        )
        print("下载完成！")
        EOF
        )

        # 导出环境变量供 Python 使用
        export LOCAL_DIR="$LOCAL_DIR"
        export DATASET_NAME="$DATASET_NAME"

        # 执行下载
        echo "开始从 Hugging Face Hub 下载数据集..."
        python3 -c "$PY_SCRIPT"

        # 检查是否成功并创建标记文件
        if [ $? -eq 0 ]; then
            touch "$LOCAL_DIR/.hf_download_complete"
            echo "数据集已保存至: $LOCAL_DIR"
        else
            echo "下载失败。请检查数据集 ID、网络或访问权限。"
            exit 1
        fi
      jobSpec:
        image: registry.baidubce.com/aihcp-public/pytorch:2.7.0-cu12.8.61-py3.12-ubuntu24.04
        replicas: 1
        envs:
          - name: DATASET_NAME
            value: '{{inputs.parameters.dataset_name}}'
      datasources:
        - type: bos
          name: ""
          sourcePath: '{{inputs.parameters.target_data_dir}}'
          mountPath: /mnt/bos/data

tasks:
  - name: job-1
    taskTemplateName: template-dataset-download-huggingface
    inputs:
      - name: queue_id
        value: aihcq-h1plvp
      - name: dataset_name
        value: liucong/Chinese-DeepSeek-R1-Distill-data-110k
      - name: target_data_dir
        value: bos://my-bucket/datasets
```

### 数据转储

#### 从BOS拉取数据到PFS

可用于从BOS动态加载冷数据到PFS上用于训练、数据处理等

参数说明：

|参数名称|说明|
|-|-|
|queue_id|队列ID，示例：aihcq-h1plvp|
|bos_path|BOS源路径，示例：bos://my-bucket/datasets/|
|bos_ak|访问BOS的ak|
|bos_sk|访问BOS的sk|
|pfs_path|PFS源路径，示例：/datasets/my-dataset|
|pfs_id|PFS实例ID|

```yaml
version: v1
kind: PipelineTemplate
taskTemplates:
  - name: bos-to-pfs
    type: CustomTask
    inputs:
      - name: queue_id
        type: string
        hint: 队列ID
      - name: bos_path
        type: string
        hint: 数据集的BOS路径
      - name: bos_ak
        type: string
        hint: BOS存储桶的访问ak
      - name: bos_sk
        type: string
        hint: BOS存储桶的访问sk
      - name: pfs_path
        type: string
        hint: PFS源路径
      - name: pfs_id
        type: string
        hint: PFS实例ID
    spec:
      queue: '{{inputs.parameters.queue_id}}'
      jobType: PyTorchJob
      command: |
        WORK_DIR="$(pwd)"
        ZIP_URL="https://doc.bce.baidu.com/bos-optimization/mac-bcecmd-0.5.10.zip"
        ZIP_FILE="mac-bcecmd-0.5.10.zip"
        EXTRACT_DIR="mac-bcecmd-0.5.10"
        echo "正在下载 bcecmd 工具..."
        curl -LO "$ZIP_URL"
        if [ $? -ne 0 ]; then
            echo "下载失败！请检查网络或 URL。"
            exit 1
        fi
        echo "正在解压..."
        unzip -o "$ZIP_FILE"
        if [ $? -ne 0 ]; then
            echo "解压失败！"
            exit 1
        fi
        cd "$EXTRACT_DIR" || { echo " 无法进入目录 $EXTRACT_DIR"; exit 1; }
        echo " 正在创建 credentials 文件..."
        mkdir -p /root/.go-bcecli
        cat > ~/.go-bcecli/credentials <<EOF
        [Defaults]
        Ak = "$BOS_AK"
        Sk = "$BOS_SK"
        EOF
        chmod 600 ~/.go-bcecli/credentials
        echo "bcecmd工具安装及配置完成,开始下载数据"
        ./bcecmd bos sync {{inputs.parameters.bos_path}} /mnt/cluster/dataset
        echo "数据集下载成功，已保存到PFS的{{inputs.parameters.pfs_path}}路径"
      jobSpec:
        image: registry.baidubce.com/inference/aibox-ubuntu:v2.0-22.04
        replicas: 1
        envs:
          - name: BOS_AK
            value: '{{inputs.parameters.bos_ak}}'
          - name: BOS_SK
            value: '{{inputs.parameters.bos_sk}}'
      datasources: 
        - type: pfs
          name: '{{inputs.parameters.pfs_id}}'
          sourcePath: '{{inputs.parameters.pfs_path}}'
          mountPath: /mnt/cluster/dataset
tasks:
  - name: sync-bos-to-pfs
    taskTemplateName: bos-to-pfs
    inputs:
      - name: queue_id
        value: aihcq-h1plvpzb5gh0
      - name: bos_path
        value: bos://my-bucket/datasets/
      - name: bos_ak
        value: <你的sk>
      - name: bos_sk
        value: <你的sk>
      - name: pfs_path
        value: /datasets/my-dataset
      - name: pfs_id
        value: pfs-xxxx
```
