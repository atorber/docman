以下以将json格式的数据转换为jsonl格式为例讲解如何进行处理。


假设有批量的json格式的数据文件需要批量转换为jsonl格式，将存储在BOS的test-a存储桶的/json目录中的所有json文件批量转换为同名的jsonl格式并保存在PFS的/jsonl目录。


原始数据格式：

```json
[
{"src": "www.nvidia.com", "text": "The quick brown fox", "type": "Eng", "id": "0", "title": "First Part"},
{"src": "The Internet", "text": "jumps over the lazy dog", "type": "Eng", "id": "42", "title": "Second Part"}
]
```

## 编写数据处理脚本

首先，需要编写数据处理脚本，支持使用任意编程语言编写转换脚本，我们以python编写为例。

创建json2jsonl.py、requirements.txt两个文件并保存在同一目录下。

json2jsonl.py文件：

```python
#!/usr/bin/env python3
"""
JSON到JSONL批量转换脚本
将JSON数组文件转换为JSONL格式（每行一个JSON对象）
"""

import argparse
import json
import sys
from pathlib import Path


def convert_json_to_jsonl(input_file: Path, output_file: Path) -> bool:
    """转换单个JSON文件为JSONL格式"""
    try:
        with open(input_file, 'r', encoding='utf-8') as f:
            data = json.load(f)
        
        if not isinstance(data, list):
            print(f"错误: {input_file.name} 不是数组格式")
            return False
        
        output_file.parent.mkdir(parents=True, exist_ok=True)
        
        with open(output_file, 'w', encoding='utf-8') as f:
            for item in data:
                if isinstance(item, dict):
                    f.write(json.dumps(item, ensure_ascii=False) + '\n')
        
        print(f"转换成功: {input_file.name} -> {output_file.name}")
        return True
        
    except Exception as e:
        print(f"转换失败 {input_file.name}: {e}")
        return False


def main():
    parser = argparse.ArgumentParser(description="JSON到JSONL批量转换")
    parser.add_argument('--input-dir', '-i', required=True, help='输入目录')
    parser.add_argument('--output-dir', '-o', required=True, help='输出目录')
    parser.add_argument('--verbose', '-v', action='store_true', help='详细日志')
    args = parser.parse_args()
    
    input_path = Path(args.input_dir)
    output_path = Path(args.output_dir)
    
    if not input_path.exists():
        print(f"错误: 输入目录不存在 {input_path}")
        sys.exit(1)
    
    json_files = list(input_path.rglob("*.json"))
    if not json_files:
        print(f"警告: 在 {input_path} 中未找到JSON文件")
        return
    
    print(f"找到 {len(json_files)} 个JSON文件")
    
    success = 0
    for json_file in json_files:
        relative_path = json_file.relative_to(input_path)
        output_file = output_path / relative_path.with_suffix('.jsonl')
        
        if convert_json_to_jsonl(json_file, output_file):
            success += 1
    
    print(f"转换完成: {success}/{len(json_files)} 个文件成功")


if __name__ == "__main__":
    main()
```

requirements.txt文件：

```txt
ujson>=5.0.0
orjson>=3.0.0
```

> 此示例中没有外部依赖库，requirements.txt中的依赖仅用于演示

## 编写启动命令

```bash
#!/bin/bash

# JSON到JSONL批量转换脚本启动器
set -e

cd /mnt/scripts/

# 默认值
INPUT_DIR="/mnt/data/json"
OUTPUT_DIR="/mnt/data/jsonl"
VERBOSE="${VERBOSE:-false}"

# 解析参数
while [[ $# -gt 0 ]]; do
    case $1 in
        -h|--help)
            echo "用法: $0 [-i INPUT_DIR] [-o OUTPUT_DIR] [-v]"
            echo "环境变量: INPUT_DIR, OUTPUT_DIR, VERBOSE"
            exit 0
            ;;
        -v|--verbose) VERBOSE="true"; shift ;;
        -i|--input-dir) INPUT_DIR="$2"; shift 2 ;;
        -o|--output-dir) OUTPUT_DIR="$2"; shift 2 ;;
        *) echo "未知参数: $1"; exit 1 ;;
    esac
done

# 安装依赖
pip install -r requirements.txt

# 检查Python
if ! command -v python3 &> /dev/null; then
    echo "错误: Python3 未安装"
    exit 1
fi

# 检查输入目录
if [ ! -d "$INPUT_DIR" ]; then
    echo "错误: 输入目录不存在: $INPUT_DIR"
    exit 1
fi

# 创建输出目录
mkdir -p "$OUTPUT_DIR"

# 构建命令
CMD="python3 json2jsonl.py --input-dir \"$INPUT_DIR\" --output-dir \"$OUTPUT_DIR\""
[ "$VERBOSE" = "true" ] && CMD="$CMD --verbose"

# 执行转换
echo "转换: $INPUT_DIR -> $OUTPUT_DIR"
eval $CMD
```

## 高级拓展
1. 业务中常用的周期执行数据处理任务可以使用百舸OpenAPI进行任务提交，也可以集成到自有系统中进行业务调用
2. 常用的处理工具打包成专门的数据处理镜像，可以避免每次上传处理脚本，直接在启动命令中调用
3. 程序处理脚本程序也可以在启动命令中使用git命令从代码库中拉取或者wget从远程服务器下载
4. 可以通过使用环境变量传递密钥、路径等信息进一步保证敏感信息安全
