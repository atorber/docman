# DocMan：产品定位与价值

## 一句话介绍

DocMan 是面向产品团队的文档智能工作台，用统一流程连接“帮助文档诊断、帮助文档生成、PRD 生成、PRD 评审”四类核心任务。

## 为谁而生

- 产品经理：在需求进入研发前完成多视角质量把关
- 研发与测试：更早识别实现复杂度、测试覆盖与风险点
- 文档团队：持续保障文档与系统一致、结构清晰、表达准确

## 解决什么问题

- 文档质量难量化：过时、错漏、冲突、失效链接难以及时发现
- 需求评审不体系化：评审标准不统一，风险识别依赖个人经验
- 结果分散难追踪：诊断、生成、评审记录分布在不同位置，复盘成本高

## 核心价值

- 统一入口：同一 Web 界面覆盖文档与 PRD 全流程任务
- 统一标准：内置结构化维度与视角，输出可复用、可复盘
- 统一记录：最近记录聚合展示，支持按来源筛选与跳转

## 工作方式

DocMan 采用“参数配置 + Prompt 生成 + 本地 Agent 执行 Skill + 结果回看”的闭环模式，兼顾低接入成本与高灵活性。
