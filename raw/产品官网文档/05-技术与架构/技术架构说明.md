# 技术架构说明

## 架构概览

DocMan 采用前后端分离 + 本地 Skill 执行模式：

- 前端 `web/`：负责参数配置、结果展示、记录跳转
- 后端 `backend/`：负责文件读取、Prompt 生成、记录聚合
- Skill `skills/`：由本地 Agent 执行实际诊断/生成/评审任务

## 后端能力边界

后端以 `/api` 为统一入口，按业务分层：

- `/api/documents`：帮助文档树与内容读取
- `/api/diagnoses`：诊断历史、报告、timeline、修复文档读写
- `/api/prompt`：诊断/帮助文档生成/PRD 生成 Prompt 与选项
- `/api/prd-review`：PRD 树、评审历史、维度视角、评审 Prompt
- `/api/records`：最近记录聚合

同时提供 `/health` 健康检查接口。

## 前端页面组织

- 统一路由入口在 `web/src/App.tsx`
- `Home` 页面承载文档诊断、最近记录、帮助文档生成、PRD 生成导航
- `PrdReview` 页面承载 PRD 评审完整流程
- API 请求统一收敛在 `web/src/services/api.ts`

## Prompt 引擎

- `promptService`：诊断、帮助文档生成、PRD 生成 Prompt 编排
- `prdPromptService`：PRD 评审视角/维度定义与 Prompt 编排
- 所有任务均带时间戳，方便文件归档与历史追踪
