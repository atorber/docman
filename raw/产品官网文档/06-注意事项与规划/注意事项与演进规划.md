# 注意事项与演进规划

## 使用注意事项

### 文档与路径约定

- 待诊断帮助文档统一放置在 `raw/`
- PRD 相关文档统一放置在 `prd/`
- PRD 生成输出路径必须是 `prd/` 下的 `.md` 文件

### 执行与协作建议

- 涉及系统一致性校验前，建议先完成目标系统登录
- 敏感信息问题（如密码、密钥、Token）应作为高优先级处理
- 评审结论用于辅助决策，最终结论建议由跨角色共同确认

### 结果管理

- 文档诊断结果重点关注：`report/`、`timeline/`、`new/`
- PRD 评审结果重点关注：`report/prd/`、`timeline/prd/`
- 建议结合“最近记录”进行阶段性复盘

## 演进规划

### 文档诊断方向

- 支持批量诊断与多文档任务编排
- 提供进度实时反馈与统计报表能力
- 持续扩展自动化检测能力与规则库

### PRD 评审方向

- 支持评审模板定制与场景化复用
- 支持多 PRD 关联评审与联动分析
- 支持评审意见协作与闭环追踪
