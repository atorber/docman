## 数据集动态加载

以下工作流实现BOS冷数据动态加载至PFS并在训练后自动清理。

![image.png](https://bce.bdstatic.com/doc/bce-doc/AIHC/image_0e6998f.png)

### 简介

**背景**
在模型训练场景中，由于训练程序需要极高的磁盘I/O吞吐，通常建议将训练数据集放置在并行文件存储PFS中以保障训练效率。而PFS的存储成本相对较高，不适合作为数据的长期持久化存储。因此，企业通常将海量数据集（冷数据）存放在成本更低的对象存储（BOS）中。

**痛点**

1. **人工介入成本高**：算法工程师在每次启动训练前，需要手动启动数据搬运任务将BOS的数据拷贝到PFS，待拷贝完成后再手动启动训练任务。
2. **资源浪费严重**：训练任务结束后，经常会忘记清理PFS上的数据集，导致昂贵的高性能存储空间被无效占用，产生不必要的成本。

**方案说明**

借助百舸**工作流，我们可以构建一个自动化的三步串行流水线**：

1. **数据预热节点**：挂载PFS，利用CPU节点将BOS中的冷数据自动拷贝至PFS中。
2. **模型训练节点**：依赖数据预热节点，待数据拷贝完成后，挂载数据预热节点相同PFS源路径执行模型训练任务。
3. **checkpoint存档及数据清理（并行执行）：**
    1. **checkpoint存档节点**：将训练得到的权重上传到bos存档。
    2. **数据清理节点**：依赖模型训练节点，待训练完成后，清理PFS上的数据集缓存，从而释放高性能存储空间。

### YAML 配置

以下是该最佳实践的完整工作流 YAML 配置。您可以将其直接复制并根据实际环境（如队列ID、PFS实例ID、BOS路径）进行替换使用。

version: v1
kind: PipelineTemplate

  - name: bos-to-pfs
    type: CustomTask
    inputs:
      - name: queue_id
        type: string
        hint: 队列ID
      - name: bos_path
        type: string
        hint: 数据集的BOS路径
      - name: bos_ak
        type: string
        hint: BOS存储桶的访问ak
      - name: bos_sk
        type: string
        hint: BOS存储桶的访问sk
      - name: pfs_path
        type: string
        hint: PFS源路径
      - name: pfs_id
        type: string
        hint: PFS实例ID
    spec:
      queue: '{{inputs.parameters.queue_id}}'
      jobType: PyTorchJob
      command: >
        WORK_DIR="$(pwd)"

        ZIP_URL="https://doc.bce.baidu.com/bos-optimization/mac-bcecmd-0.5.10.zip"

        ZIP_FILE="mac-bcecmd-0.5.10.zip"

        EXTRACT_DIR="mac-bcecmd-0.5.10"

        echo "正在下载 bcecmd 工具..."

        curl -LO "$ZIP_URL"

        if [ $? -ne 0 ]; then
            echo "下载失败！请检查网络或 URL。"
            exit 1
        fi

        echo "正在解压..."

        unzip -o "$ZIP_FILE"

        if [ $? -ne 0 ]; then
            echo "解压失败！"
            exit 1
        fi

        cd "$EXTRACT_DIR" || { echo " 无法进入目录 $EXTRACT_DIR"; exit 1; }

        echo " 正在创建 credentials 文件..."

        mkdir -p /root/.go-bcecli

        cat > ~/.go-bcecli/credentials <<EOF

        [Defaults]

        Ak = "$BOS_AK"

        Sk = "$BOS_SK"

        EOF

        chmod 600 ~/.go-bcecli/credentials

        echo "bcecmd工具安装及配置完成,开始下载数据"

        ./bcecmd bos sync {{inputs.parameters.bos_path}} /mnt/cluster/dataset

        echo "数据集下载成功，已保存到PFS的{{inputs.parameters.pfs_path}}路径"
      jobSpec:

      - name: pfs_id
        type: string
        hint: PFS实例ID
    spec:
      queue: '{{inputs.parameters.queue_id}}'

      command: >
        WORK_DIR="$(pwd)"

        ZIP_FILE="mac-bcecmd-0.5.10.zip"

        EXTRACT_DIR="mac-bcecmd-0.5.10"

        echo "正在下载 bcecmd 工具..."

        curl -LO "$ZIP_URL"

        if [ $? -ne 0 ]; then
            echo "下载失败！请检查网络或 URL。"
            exit 1
        fi

        echo "正在解压..."

        EOF

      priority: normal
