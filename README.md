# 文档诊断工具

一个面向产品帮助文档的智能诊断系统，帮助发现文档中的各种质量问题。

## 产品介绍

### 背景与问题

在日常的文档维护过程中，我们经常遇到以下问题：

- **文档过时**：系统UI已经更新，但文档截图和描述还停留在旧版本
- **内容错误**：语法错误、错别字、描述不清晰等问题影响阅读体验
- **流程缺失**：操作步骤不完整，新用户按照文档操作可能失败
- **信息不一致**：文档各章节描述前后矛盾，术语使用不统一
- **维护困难**：大量文档缺乏系统性的质量检查手段，问题发现依赖用户反馈

### 解决方案

本项目是一个**文档诊断工具**，通过自动化的方式对产品帮助文档进行全方位质量检查，帮助文档维护者快速发现并修复问题。

### 核心能力

1. **18维度全面诊断**
   - 文档与系统一致性检查
   - 语法错误和错别字检测
   - 与常识违背和前后文冲突检查
   - 过时信息和失效链接检测
   - 敏感信息泄露扫描
   - 代码示例错误检查
   - 截图完整性验证
   - 格式一致性检查
   - 文档完整性评估
   - 以及更多...

2. **与实际系统对比验证**
   - 支持指定目标系统URL
   - 自动打开浏览器进行界面一致性检查
   - 验证菜单、按钮、文案等是否与实际系统一致

3. **可视化诊断历史管理**
   - Web界面直观展示文档目录结构
   - 完整的诊断历史记录
   - 支持查看每次诊断的详细信息（报告、timeline、修复后文档）

4. **一键生成诊断Prompt**
   - 配置诊断参数（目标URL、自定义检查要求、诊断维度）
   - 自动生成完整的诊断Prompt
   - 可直接在Comate中执行诊断

### 适用场景

- **文档维护**：定期对产品文档进行质量检查
- **版本发布**：新版本上线前检查文档一致性
- **新文档审核**：对新编写的文档进行质量把关
- **问题排查**：用户反馈文档问题时快速定位

## 快速开始

### 环境要求

- Node.js >= 18.0
- npm >= 8.0

### 安装依赖

```bash
# 安装后端依赖
cd backend
npm install

# 安装前端依赖
cd ../web
npm install
```

### 启动服务

```bash
# 启动后端服务 (端口 3001)
cd backend
npm run dev

# 启动前端服务 (端口 3000)
cd web
npm run dev
```

### 使用流程

1. **打开Web界面**
   访问 http://localhost:3000

2. **选择文档**
   在左侧文档目录树中选择需要诊断的文档

3. **查看诊断历史**
   中间面板显示该文档的历史诊断记录

4. **生成诊断Prompt**
   点击右上角"生成诊断Prompt"按钮，配置诊断参数后生成Prompt

5. **执行诊断**
   复制生成的Prompt，在Comate中执行诊断

6. **查看结果**
   刷新页面，查看诊断报告、修复后的文档和详细过程记录

## 诊断标准说明

### 高优先级问题
- 敏感信息泄露（密码、密钥、Token等）
- 失效链接/图片
- 与系统不一致
- 代码示例错误
- 缺少必要的警告/提示

### 中优先级问题
- 语法错误
- 错别字
- 前后文冲突
- 过时信息
- 缺失步骤
- 版本兼容性问题

### 低优先级问题
- 格式不一致
- 描述不清晰
- 截图缺失
- 冗余内容
- 命名不规范
- 文档完整性问题

## 注意事项

1. 原始文档需要放在 `raw/` 目录下
2. 诊断过程中可能需要登录目标系统，请提前准备好账号
3. 敏感信息泄露检测是强制高优先级，任何密码、密钥等敏感词都会高亮提醒
4. 建议定期进行文档诊断，保持文档质量
