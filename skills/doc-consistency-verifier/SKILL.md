# 文档与系统一致性验证工具

本skill用于验证文档描述与实际系统界面的一致性，帮助发现文档过时或描述不准确的问题。

## 适用场景

- 验证操作手册是否与实际系统一致
- 检查产品文档的UI截图是否过时
- 验证菜单路径、操作流程的准确性
- 核对按钮文案、提示信息等细节

## 验证标准

一致性验证包含以下4个维度：

1. **菜单/入口一致性**：左侧菜单结构、导航路径、入口位置
2. **操作流程一致性**：操作步骤的顺序和完整性
3. **UI/界面一致性**：页面布局、组件样式、截图对比
4. **按钮/文案一致性**：按钮名称、提示文案、错误信息等

## 使用前提

**使用此skill时，用户必须提供以下信息**：

1. **要验证的文档**：文档的完整路径
2. **对应操作界面网址**：需要验证的系统URL

如果用户未提供上述信息，询问用户补充后再继续执行。

## 验证流程

### 步骤1：准备工作

1. 明确验证目标：
   - 目标系统URL
   - 需要验证的文档路径
   - 验证的具体功能点

2. 检查文档是否存在：
   - 如果文档不存在，告知用户并终止任务

3. 准备输出目录：
   - 在原文档相同目录下创建输出文件
   - 如果用户指定了自定义输出位置，使用用户指定的位置

### 步骤2：打开浏览器访问目标页面

使用 `browser-use` skill 打开目标系统URL：

1. 首先调用 browser-use skill
2. 在 browser-use 中执行：导航到用户提供的URL
3. 检查登录状态：
   - 如果需要登录，提示用户手动登录
   - 等待用户登录完成后继续执行验证

### 步骤3：执行一致性验证

按照验证标准逐项对比文档与实际系统：

#### 3.1 菜单/入口验证

- 记录实际系统的左侧菜单结构
- 对比文档中的菜单描述和截图
- 检查菜单项名称、顺序、分组是否一致

#### 3.2 操作流程验证

- 记录实际系统的操作步骤
- 对比文档描述的操作流程
- 检查步骤顺序、按钮位置、交互逻辑

#### 3.3 UI/界面验证

- 截图记录关键页面
- 对比文档中的截图
- 检查页面布局、组件样式

#### 3.4 按钮/文案验证

- 记录实际的按钮文字、提示信息
- 对比文档中的描述
- 检查是否有差异

### 步骤4：生成验证报告

验证完成后，生成不一致点报告，格式如下：

```markdown
# 文档验证报告：[文档名称]

**验证日期**：YYYY-MM-DD
**验证目标**：对照实际系统验证文档描述和截图的一致性
**验证范围**：菜单/入口一致性、操作流程一致性、UI/界面一致性、按钮/文案一致性
```

### 步骤5：创建更新后的文档

基于验证结果，创建更新后的新文档：

1. 在原文档相同目录下创建新文档
2. 文件名格式：`[原文档名]new.md`
3. 修复验证中发现的不一致问题
4. 更新过时的截图

## 输出要求

1. **验证报告**：详细记录所有不一致点
2. **更新后的文档**：基于原文档修复不一致内容
3. 如果需要，输出修复建议

## 注意事项

- 如果页面需要登录，先提示用户登录
- 截图是验证UI一致性的重要依据
- 优先关注影响用户操作的问题
- 保持报告客观、准确
