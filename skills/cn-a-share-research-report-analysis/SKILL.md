# A股研报分析（外部对比驱动）

## 适用场景

- 用户提供券商/机构研报（深度/点评/更新/电话会纪要整理版）
- 用户需要快速判断：结论是否扎实、假设是否可验证、风险收益是否匹配
- 用户需要从研报中**挖掘具备外证支撑的上涨潜力线索**、并**识别外证与逻辑上站得住脚的潜在风险标的**（输出为研究线索表，非荐股）
- 默认强约束：
  - 公司最近 **3 期**财务与经营趋势交叉验证
  - 同行业可比公司 **分位**对比（盈利、估值、质量）
  - 关键事件与公告 **外部佐证**

## Agent 深度研究协议（优先执行，与 Step 2 绑定）

本 skill 的「深度」体现在 **外证充分性**，而非篇幅。当运行环境具备更强 Agent 能力时，**禁止**用单轮通读研报 + 一次笼统检索替代下列协议；能力缺失时按「降级」执行，仍须覆盖全部外证项。

### 能力映射（有则必用）

| 能力 | 用法 |
| --- | --- |
| **Plan / 计划模式** | 完成 Step 1 后、进入大规模外证前，先输出「可验证假设清单 + 待检索子问题」（按 2.1 / 2.2 / 2.3 拆条），再执行外证，避免漏项。 |
| **Task / 子代理（如 explore）** | 将 Step 2 **拆成并行工作流**：线程 A=3 期趋势与财务口径；线程 B=行业分位与可比池；线程 C=公告/新闻/政策事件佐证。合并前做 **矛盾点交叉核对**。 |
| **联网检索（多轮）** | 首轮按「东方财富」等优先级站点拉齐数据口径；**第二轮**仅针对研报与首轮数据的 **差异、断层、时点** 追加查询，直至逐项标注「一致 / 部分一致 / 未核验」。 |

### 强制加深触发（满足任一条即须多轮检索，不得单次收束）

- 研报含 **目标价、评级上调/下调、首次覆盖**；
- **并购、重组、定增、股权激励、业绩预告/快报、重大订单** 等里程碑叙事；
- **政策敏感**行业（限产、调价、牌照、出海制裁等）或宏观表述占比较高；
- 用户 `分析偏好` 显式包含「深度」「验证」「较真」「外证」。
- `标的挖掘侧重点` 为 `偏重潜力标的` 或 `偏重风险警示` 时：对应一侧须增加叙事深度与分项证据，另一侧不得留空（至少「无符合条件」一行说明）。

### 降级（无 Plan/Task/仅能单次搜索时）

- **串行**完成 2.1 → 2.2 → 2.3，且主报告「外部对比完成情况」中注明：未使用并行子任务 / 未做多轮检索（如实填写）。
- 仍须满足置信度降级规则与高/中/低标注。

## 重点关注框架（强制，按报告类型分流）

每次执行都必须覆盖以下 8 项，并形成“结论 + 证据 + 置信度”：

1. 核心结论与评级变化（评级/目标价/观点方向变化）
2. 关键假设是否成立（量、价、份额、成本、费用等）
3. 最近3期趋势验证（营收、利润、现金流、负债是否支持观点）
4. 估值框架合理性（PE/PB/PS/DCF 与可比口径是否一致）
5. 催化剂与时间表（是否可验证、触发条件是否清晰）
6. 风险披露充分性（是否遗漏关键反证或下行变量）
7. 外部事件佐证一致性（公告/产业数据/政策是否支持叙事）
8. 跟踪指标清单（未来1-2期必须复核的KPI）

> 说明：8 项框架为通用骨架，但“取证口径与权重”必须按报告类型分流（个股研究 vs 行业板块研究）。

## 输入参数（prompt 提供）

| 参数 | 必填 | 说明 |
| --- | --- | --- |
| 原始文档路径 | 条件必填 | 相对项目根目录路径，通常位于 `research_reports/` |
| 原始文档文本 | 条件必填 | 直接粘贴研报正文或关键段落 |
| 报告类型 | ✅ | `个股研究` 或 `行业板块研究`（若未提供则先自动识别，识别失败需补问） |
| 时间戳 | ✅ | `YYYYMMDD_HHMMSS`（输出命名） |
| 外部数据优先级 | ❌ | 默认 `东方财富` |
| 分析偏好 | ❌ | 如 `偏风险` / `偏机会` / `偏估值` / `偏业绩兑现` |
| 标的挖掘侧重点 | ❌ | 默认 `潜力与风险兼顾`；可选 `偏重潜力标的` / `偏重风险警示` |

约束：`原始文档路径` 与 `原始文档文本` 至少提供一项；若同时提供，优先文档路径。

## 潜力标的与风险标的：判定口径（强制）

> **性质说明**：本节依据「研报论据 + Step 2 外部三件套」做**可追溯的归类**，用于研究工作流；**不构成**买卖或持仓建议；最终决策须用户自行判断。

### 潜力标的（「值得关注」线索）

在研报涉及的公司/板块成分中，满足 **越多项** 且外证越一致，越可标为 **潜力跟踪**（用 `高 / 中 / 低` 表示**外证支撑强度**，非涨幅承诺）：

1. **逻辑**：研报给出清晰看多链条（景气、份额、价格、订单、新产品、政策受益等），且与近期 **3 期趋势**同向或拐点可解释。
2. **位置**：相对同业 **分位/估值** 未明显透支（或研报已给出可核对的估值锚，且外证口径下仍合理）；若无法取得分位，须标注「估值外证不足」并整体降档。
3. **催化剂**：未来 1～2 个季度内存在可观测 KPI 或事件；且 **2.3 事件佐证** 为「一致」或「部分一致」。
4. **风险对称**：研报与原文已提示的主要风险，在外证中 **尚未** 出现明确证伪信号（若已出现则不得列入「潜力跟踪」，应转入风险池或标为「逻辑存疑」）。

**个股研报**：至少输出 1 行「潜力侧」结论（若外证全面背离，则潜力标的一栏明确写 **无**：主逻辑被外证削弱的原因）。

**行业/板块研报**：从研报中抽取 **全部被明确点评或首推/备选覆盖的标的**，逐只填「潜力跟踪池」表；顺带提及且无实质论据的标的可单列「轻仓观察/略提」且不强行标潜力等级。

### 风险标的（「风险警示」线索）

下列情形任一条成立，须将对应标的纳入 **风险提示池**（可与「潜力跟踪」同日并存于不同维度：基本面风险 vs 交易拥挤等）：

1. **外证背离**：3 期趋势、现金流或杠杆与研报核心假设 **明显矛盾**。
2. **事件负面**：公告、监管、诉讼、减值、大额解禁、质押、减持等外证已出现，而研报未充分消化或仍维持过度乐观表述。
3. **估值透支**：同业分位与增速/ROE 不匹配，且研报目标价依赖过强假设。
4. **脆弱性**：客户/订单集中、政策敏感、财务口径争议、治理瑕疵等，外证可部分印证。

**输出要求**：风险池须写清 **风险类型**（基本面 / 估值 / 事件 / 流动性与筹码 / 政策与合规）与 **严重度预估**（高/中/低）及一项 **下期复核指标**。

### 与子代理并行（可选）

研报覆盖 **≥3 只可交易标的** 时：在 Task 并行分配中优先为 **每只核心标的**切一条「趋势 + 分位 + 关键公告」查证线，再在主报告中汇入潜力/风险两表。

## 输出配置（固定默认）

- 主报告：`report/research/cn-a-share/{股票代码}/{报告期}_{时间戳}_report.md`
- 过程记录（建议）：`timeline/research/cn-a-share/{股票代码}/{报告期}_{时间戳}_timeline.json`

## 极简输出协议（强制）

> 目标：聚焦可决策信息，避免长篇叙事。

1. 只写结论性信息：每条都要有 `结论 + 证据 + 置信度`，不写背景铺垫。
2. 严格控制篇幅：
   - 「一句话结论」不超过 **80 字**；
   - 「看多逻辑 / 外证信号 / 风险说明」单格不超过 **40 字**；
   - 全文除表格外，单段不超过 **3 行**。
3. 严格控制数量：
   - 核心观点最多 **5 条**；
   - 叙事性要点最多 **3 条风险 + 3 条机会**；
   - 跟踪清单最多 **6 条 KPI**。
4. 禁止重复：同一证据不得在不同章节重复长写；若已在表格写明，正文只做一句引用。
5. 信息不足时直接写「未核验/缺失原因」，不使用泛化措辞补长文本。

## 执行流程（必须按顺序）

### Step 1：输入归一化与证据抽取

1. 输入归一化（必须）
   - 有 `原始文档路径`：直接读取
   - 仅有 `原始文档文本`：先保存到 `research_reports/imported/{时间戳}_research_input.md`

2. 文档元信息抽取（必须）
   - 股票代码/简称
   - 研报类型（首次覆盖/更新/点评/电话会/行业对比）
   - 报告期（如 `2026Q1` / `2025年报`）
   - 机构与作者（如可得）
   - 评级与目标价（如可得）
   - 报告类型确认：`个股研究` / `行业板块研究`
   - 若为 `行业板块研究`：从正文抽取 **覆盖标的清单**（代码、简称、首推/备选/顺带提及等角色），供潜力跟踪池与风险提示池逐行映射

3. 核心观点结构化（必须）
   - 投资主线（1-3条）
   - 关键假设（量、价、份额、成本、费用、资本开支）
   - 催化剂与触发条件（时间、事件、指标）
   - 风险项（行业、公司、财务、治理、政策）

4. 证据摘录（必须）
   - 数字证据：收入、利润、毛利率、现金流、估值倍数等
   - 文字证据：研报中用于支持结论的关键段落

### Step 2：外部对比三件套（每次都必须请求）

> 外部数据优先级：东方财富。若部分字段失败，必须记录缺失并降置信度。  
> **执行顺序**：若环境允许并行（见「Agent 深度研究协议」），三组可并行推进后再合并；否则按 2.1 → 2.2 → 2.3 串行。**合并阶段**必须处理：口径不一致（同比/环比/回溯调整）、币种、非经常损益、分部披露差异。

#### 2.1 最近3期趋势对比（必须）

按报告频率匹配最近3期（季度/半年/年度）：

- 营收、归母净利、扣非净利（可得子集）
- 毛利率/净利率、经营现金流净额
- 应收、存货、负债结构（可得子集）

输出：每项指标的 `方向 + 拐点/背离`，并标记是否支持研报核心假设。

#### 2.2 行业分位与可比对照（必须）

外部获取行业口径并对比同业（至少3家可比，或使用同业排名口径）：

- 盈利能力：毛利率、净利率、ROE/ROA（可得子集）
- 质量与风险：资产负债率、现金流匹配度
- 估值：PE/PB/PS 或 EV 体系（可得子集）

输出：`分位/排名 + 偏离方向 + 是否支持研报结论`。

#### 2.3 事件与公告佐证（必须尝试）

核验研报中关键叙事是否被外部事件支持：

- 业绩预告/快报、订单、产能、价格、政策、并购、诉讼、减值、回购等

输出：`一致 / 部分一致 / 未核验`，并列出缺失项。

### Step 2.5：类型化分析主线（强制分流）

#### A. 个股研究（必须侧重）

- 业绩兑现：营收/利润/现金流最近3期是否验证个股逻辑
- 估值定价：目标价与估值模型（PE/PB/PS/DCF）是否可复算
- 公司竞争力：份额、产品、渠道、成本、治理
- 个股催化剂：订单、新品、并购、业绩拐点时间表
- 个股风险：减值、诉讼、客户集中、负债与再融资压力

#### B. 行业板块研究（必须侧重）

- 行业周期：渗透率、库存、价格、产能利用率阶段
- 景气验证：量价、开工率、招投标、政策节奏
- 产业链传导：上中下游利润分配与传导路径
- 板块估值与拥挤度：行业分位、交易拥挤、风格暴露
- 受益方向分层：龙头/弹性/受损标的的筛选逻辑

### Step 2.8：多公司逐一分析与投资价值排序（涉及多家公司时强制）

当研报覆盖 **2 家及以上上市公司**（含首推/重点覆盖/可比公司）时，必须逐家公司分析，不得只给行业总评。

每家公司至少输出以下信息：

1. 行业地位：龙头/二线/跟随者 + 简要依据
2. 近期表现：最近3期趋势（好转/走平/承压）
3. 估值位置：同业分位（低/中/高）与是否透支
4. 催化与风险：未来1-2期各1条关键催化与风险
5. 投资价值结论：`高 / 中 / 低`（研究价值分层）

随后必须输出「投资价值专业排序表」：

| 排名 | 标的代码 | 简称 | 投资价值等级 | 排序理由（<=40字） | 关键反证条件（<=30字） |
| --- | --- | --- | --- | --- | --- |

排序规则（必须）：

- 优先级：基本面趋势验证 > 行业地位竞争力 > 估值性价比 > 催化可兑现性 > 风险暴露度。
- 若证据冲突：保守降级，并在理由中写明冲突点。
- 若并列：允许并列，但需说明并列原因。

### Step 3：结论置信度判定（强制）

对每条核心观点打 `高 / 中 / 低`：

- 高：研报证据充分 + 3期趋势支持 + 行业分位支持 + 事件一致
- 中：研报证据充分 + 趋势支持，但分位/事件缺失或不完整
- 低：关键外部对比缺失较多，或口径无法对齐

降级规则：

- 行业分位缺失：涉及“行业领先/落后”结论降为中/低
- 事件未核验：涉及“催化剂兑现”结论降为中/低

### Step 4：潜力/风险双线结论与 Top 5（生成研究结论）

在完成 Step 2 外部三件套后，须先产出 **标的维度**结论，再提炼 Top 5 叙事观点：

- **潜力跟踪**：对研报覆盖的每只核心标的给出「外证支撑强度」等级；`标的挖掘侧重点=偏重潜力标的` 时扩充叙述与分项指标。
- **风险警示**：对每只存在外证疑点或研报乐观偏差的标的强制入表；`标的挖掘侧重点=偏重风险警示` 时增加严重度说明与证据链。

每条核心观点（Top 5）仍须包含：

- 观点主题（机会/风险/中性观察）
- 一句话结论
- 研报证据（数字/原文）
- 外部验证（3期趋势 + 行业分位 + 事件核验）
- 反证条件（什么情况下观点失效）
- 跟踪指标（下期需验证的关键KPI）
- 置信度（高/中/低）

并且必须追加一节“重点关注框架复核（8项）”，逐项输出：

- 结论：支持 / 部分支持 / 不支持
- 主要证据：文档证据 + 外部证据
- 置信度：高 / 中 / 低

并且必须追加“类型化复核清单（必填）”：

- 若 `个股研究`：输出“个股五项复核”逐条结论（支持/部分支持/不支持）
- 若 `行业板块研究`：输出“行业五项复核”逐条结论（支持/部分支持/不支持）

### Step 5：输出报告（简洁版）

报告必须包含以下结构：

1. 一句话结论（偏多/偏空/分歧 + 总体置信度；若为多标的研报，简述潜力/风险两边的整体倾向）
2. 基础信息（股票代码/简称、报告期、研报类型、报告类型、机构、评级/目标价）
3. **潜力跟踪池（必填）**：Markdown 表格，每一行对应一只「值得关注」候选标的（个股研报至少一行；行业研报覆盖几只写几只，无明显潜力则表内说明「无外证支撑标的」）。
   | 列 | 说明 |
   | --- | --- |
   | `标的代码` | 六位 |
   | `简称` | |
   | `研报角色` | 首推/备选/顺带/覆盖等 |
   | `看多逻辑（与研报对齐）` | 一句话 |
   | `外证支撑` | **高 / 中 / 低**（综合 3 期趋势 + 分位合理 + 事件一致） |
   | `验证缺口或下期 KPI` | |
   | `入选值得关注` | **是 / 否**（「否」须写简短原因） |
4. **风险提示池（必填）**：Markdown 表格，每一处「风险警示」线索一行。
   | 列 | 说明 |
   | --- | --- |
