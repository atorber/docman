# 帮助文档生成工具

本skill用于根据产品需求文档（PRD）和控制台实际界面，自动生成专业的产品帮助文档。

## 适用场景

- 根据PRD文档生成操作手册
- 创建用户指南和功能说明文档
- 快速入门文档自动生成
- API使用说明文档生成

## 调用方式

用户通过prompt提供以下参数：

| 参数 | 格式 | 必填 | 说明 |
|------|------|------|------|
| PRD文档路径 | `prd/xxx.md` 或 `prd/xxx.docx` | ✅ | **工作目录内**PRD文档路径（支持.md/.docx/.txt/.pdf） |
| 生成文档路径（输出） | `raw/操作指南/xxx.md`（或相对 `raw` 的路径） | ❌ | 生成后的帮助文档输出位置 |
| 控制台URL | `https://...` | ✅ | 产品控制台的入口地址 |
| 产品名称 | `xxx平台` | ❌ | 用于生成文档标题（可选） |
| 文档类型 | `快速入门/操作指南/功能说明` | ❌ | 默认：操作指南 |
| 目标受众 | `开发者/运维人员/普通用户` | ❌ | 默认：普通用户 |
| 输出格式 | `Markdown/HTML` | ❌ | 默认：Markdown |
| 使用已登录浏览器 | `是/否` | ❌ | 是否复用已登录的浏览器窗口 |
| 显示浏览器界面 | `是/否` | ❌ | 是否显示浏览器操作界面 |
| 截图模式 | `完整页面（fullpage）/可视区域（viewport）` | ❌ | 默认完整页面。完整页面模式下不得只截取首屏 |
| 时间戳 | `YYYYMMDD_HHMMSS` | ✅ | 输出文件命名的时间戳 |

**prompt示例**：
```
# 文档生成

## 输入信息
- PRD文档路径（输入）: prd/AIHC_2.0_功能需求文档.md
- 生成文档路径（输出）: raw/操作指南/AIHC_操作指南_20260428_143052.md
- 控制台URL: https://console.bce.baidu.com/aihc
- 文档类型: 操作指南
- 目标受众: 普通用户
- 使用已登录浏览器: 是
- 显示浏览器界面: 是

## 输出配置
- 时间戳: 20260428_143052
```

## 执行流程

### 阶段一：PRD文档解析

**目标**：从PRD文档中提取功能点、操作流程、业务规则等关键信息。

#### 1.1 读取PRD文档

支持多种格式的PRD文档：
- **Markdown (.md)**：直接读取解析
- **Word (.docx)**：解析文档结构，提取文本内容
- **文本文件 (.txt)**：直接读取
- **PDF (.pdf)**：提取文本内容

#### 1.2 信息提取清单

从PRD中提取以下结构化信息：

```json
{
  "product_info": {
    "name": "产品名称",
    "version": "版本号",
    "description": "产品简介"
  },
  "feature_modules": [
    {
      "id": "F001",
      "name": "功能模块名称",
      "description": "功能描述",
      "user_scenarios": ["场景1", "场景2"],
      "operations": [
        {
          "step": 1,
          "action": "操作动作",
          "target": "目标元素",
          "expected_result": "预期结果"
        }
      ],
      "constraints": ["限制条件1", "限制条件2"],
      "permissions": "所需权限"
    }
  ],
  "user_roles": [
    {
      "role": "角色名称",
      "permissions": ["权限1", "权限2"]
    }
  ],
  "business_rules": [
    {
      "rule": "业务规则描述",
      "applies_to": "适用范围"
    }
  ],
  "ui_elements": [
    {
      "page": "页面名称",
      "elements": [
        {
          "type": "button/input/select/...",
          "name": "元素名称",
          "description": "元素说明"
        }
      ]
    }
  ]
}
```

#### 1.3 功能树构建

将提取的功能点组织成层级结构：

```
产品功能树
├── 功能模块A
│   ├── 子功能A.1
│   │   ├── 操作步骤1
│   │   └── 操作步骤2
│   └── 子功能A.2
├── 功能模块B
│   ├── 子功能B.1
│   └── 子功能B.2
└── 功能模块C
```

### 阶段二：控制台探索

**目标**：通过浏览器自动化，收集控制台实际界面信息。

#### 2.1 浏览器会话管理

**前置检查：先检查是否有活跃的浏览器会话**

```bash
# 检查活跃会话
browser-use sessions
```

**情况A：未配置使用已登录浏览器**
```bash
# 显示浏览器界面
browser-use --browser real --profile "Default" --headed open <控制台URL>

# 不显示浏览器界面
browser-use --browser real --profile "Default" open <控制台URL>
```

**情况B：配置了使用已登录浏览器**
```bash
# 复用已有会话或创建新会话
browser-use --browser real --profile "Default" --headed open <控制台URL>
```

#### 2.2 页面结构扫描

对每个功能页面执行以下操作：

1. **导航到目标页面**
   ```bash
   browser-use open <页面URL>
   browser-use wait selector "body" --timeout 5000
   ```

2. **等待页面稳定（强制）**
   - 必须在截图前等待页面稳定，不允许页面加载中直接截图
   - 至少满足以下条件后才可截图：
     - 页面主内容容器可见（如 `main`、`.content` 或业务主容器）
     - 无明显 loading/骨架屏/进度条
     - 连续两次状态检查（间隔 1-2 秒）DOM 结构无明显变化
   - 如果页面存在懒加载，先滚动到底部再回到顶部，确保内容完成渲染

3. **获取页面元素状态**
   ```bash
   browser-use state
   ```

4. **截取页面截图**
   ```bash
   browser-use screenshot {输出文档目录}/images/01_页面名称.png
   ```
   - `{输出文档目录}` 为生成文档路径（输出）所在的目录，例如输出路径为 `raw/产品官网文档/04-使用指南/操作说明.md`，则截图保存到 `raw/产品官网文档/04-使用指南/images/01_页面名称.png`
   - 若输入参数为 `截图模式: 完整页面（fullpage）`：必须优先输出完整页面截图
   - 若页面过长或工具不支持完整截图：按"顶部/中部/底部"分段滚动截图，文件名标注段位（如 `_top/_mid/_bottom`）
   - 每次分段截图前都必须等待该段视图稳定（至少 1 秒）再截图
   - 禁止仅保留首屏截图作为该页面唯一截图

5. **提取页面信息**
   - 导航菜单结构
   - 功能入口位置
   - 表单字段和验证规则
   - 按钮和操作项
   - 提示信息和帮助文案
   - 必填项标识

#### 2.3 功能模块遍历

按照PRD提取的功能模块列表，逐一访问控制台：

```bash
FOR EACH 功能模块 IN 功能模块列表:
  1. 导航到功能入口页面
  2. 截取入口截图
  3. 记录菜单路径和入口位置
  4. 进入功能详情页
  5. 扫描页面所有可交互元素
  6. 截取关键界面截图
  7. 记录操作按钮、输入字段、选项说明
  8. 收集页面提示文案和帮助信息
  9. 识别必填项和校验规则
```

#### 2.4 交互流程记录

对于需要操作的流程（如创建、编辑、删除等）：

1. **记录操作前状态**：截图保存初始界面
2. **执行操作步骤**：按PRD描述的操作流程执行
3. **记录操作结果**：截图保存操作后界面
4. **收集系统反馈**：成功/失败提示信息

**浏览器操作命令示例**：
```bash
# 点击元素
browser-use click <index>

# 输入内容
browser-use input <index> "内容"

# 选择下拉选项
browser-use select <index> "选项值"

# 等待元素
browser-use wait selector "button.primary" --timeout 5000

# 等待文本
browser-use wait text "操作成功" --timeout 10000

# 截图（{输出文档目录} 即生成文档路径所在目录）
browser-use screenshot {输出文档目录}/images/02_操作步骤.png
```

### 阶段三：文档生成

**目标**：基于PRD内容和控制台实际信息，生成结构化的帮助文档。

#### 3.1 文档结构组织

根据文档类型选择合适的结构：

**快速入门**：
```
1. 产品简介
2. 快速上手（核心功能）
3. 下一步
```

**操作指南**：
```
1. 概述
2. 前提条件
3. 功能模块列表
4. 各功能详细操作步骤
5. 常见问题
```

**功能说明**：
```
1. 功能概述
2. 功能详解
3. 使用场景
4. 限制与约束
5. API参考（如有）
```

#### 3.2 内容合成规则

**操作步骤生成**：
1. 从PRD提取操作意图和业务背景
2. 从控制台收集实际UI元素和文案
3. 结合两者生成准确的步骤描述
4. 插入对应的截图

**注意事项提取**：
- 从PRD提取限制条件和约束
- 从控制台收集校验规则和错误提示
- 汇总生成注意事项章节

**截图管理**：
- 截图按功能模块分类存储
- 文件命名：`序号_功能名称_操作描述.png`
- **截图保存路径**：`{输出文档目录}/images/`，即与生成文档同目录下的 `images/` 子目录
- 在文档中使用相对路径引用，格式为 `images/01_xxx.png`
- 完整性要求：关键页面至少覆盖顶部导航、核心内容区、页面底部（或最后一个关键组件）
- 稳定性要求：截图时不得出现明显加载占位（Skeleton/Spinner/Loading）

#### 3.3 文档风格规范

从专业产品经理视角，文档应遵循以下风格：

**语言风格**：
- 使用清晰、简洁的第二人称（"您"）
- 操作步骤使用祈使句（"点击"、"输入"、"选择"）
- 避免技术术语堆砌，必要时提供解释

**结构规范**：
- 标题层级不超过4级
- 每个章节有明确的主题
- 步骤使用有序列表
- 注意事项使用引用块

**内容要求**：
- 准确描述UI元素名称和位置
- 说明操作的前提条件和权限要求
- 提供操作结果的预期反馈
- 标注风险操作和不可逆操作

**格式统一**：
- 菜单路径使用 `>` 连接：`控制台 > 功能模块 > 子功能`
- 按钮名称使用「」包裹：点击「确定」按钮
- 必填项使用 `*` 标注
- 截图添加图注说明

- 已开通相关服务权限

## 错误处理

| 异常情况 | 处理方式 |
|----------|----------|
| PRD文档不存在 | 提示"PRD文档不存在或不在工作目录中，请检查路径"，终止任务 |
| PRD格式不支持 | 提示"不支持的文档格式，请提供.md/.docx/.txt/.pdf文件" |
| 控制台无法访问 | 提示"控制台URL无法访问"，询问是否继续生成（仅基于PRD） |
| 需要登录 | 提示用户登录或使用已登录浏览器选项 |
| 功能入口找不到 | 记录到文档的"已知限制"章节，继续处理其他功能 |
| 页面加载超时 | 等待重试，超过3次则跳过该功能模块 |

## 质量检查

文档生成后，自动执行以下检查：

1. **完整性检查**：确保所有PRD中的功能模块都有对应章节
2. **截图检查**：确保所有引用的截图文件存在
3. **链接检查**：确保控制台URL有效
4. **格式检查**：确保Markdown语法正确
5. **术语一致性**：确保产品名称、功能名称在全文中一致

## 注意事项

- PRD文档应包含完整的功能描述和操作流程
- 控制台需要可访问且账号有足够权限
- 复杂功能可能需要多次截图才能完整展示
- 生成的文档建议人工审核后发布
- 如控制台有更新，建议重新生成文档

## 与其他Skill的协作

- **browser-use**：用于浏览器自动化操作和截图
- **doc-consistency-verifier**：生成后可用于验证文档与系统一致性
