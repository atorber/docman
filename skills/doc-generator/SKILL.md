# 帮助文档生成工具（精简版）

目标：基于 PRD 与控制台实际界面，生成高质量、可落地的帮助文档。

## 输入参数

| 参数 | 必填 | 说明 |
|---|---|---|
| PRD文档路径 | ✅ | `prd/` 下文档，支持 `.md/.docx/.txt/.pdf` |
| 控制台URL | ✅ | 待探索产品控制台入口 |
| 时间戳 | ✅ | 输出标识 |
| 生成文档路径（输出） | ❌ | 默认由调用方给出，建议 `raw/.../*.md` |
| 文档类型 | ❌ | `快速入门/操作指南/功能说明`，默认 `操作指南` |
| 目标受众 | ❌ | `普通用户/开发者/运维人员`，默认 `普通用户` |
| 输出格式 | ❌ | `Markdown/HTML`，默认 `Markdown` |
| 使用已登录浏览器 | ❌ | 复用登录态 |
| 显示浏览器界面 | ❌ | 便于观察执行过程 |
| 截图模式 | ❌ | `fullpage/viewport`，默认 `fullpage` |

## 执行流程

### 1) PRD 解析
- 读取并抽取：产品信息、功能模块、操作流程、约束规则、术语。
- 产出“功能清单 + 关键流程列表”，作为后续探索与成文骨架。

### 2) 控制台探索与截图（强约束）
- 进入目标页面后，先等待稳定再截图。
- 截图前必须满足：
  - 主内容可见，无明显 loading/骨架屏；
  - 连续两次状态检查（间隔 1-2 秒）无明显变化；
  - 若为双栏页面：`.ant-layout-sider` 可见，且出现“帮助文档目录”或“PRD文档目录”，左栏宽度 >= 200px。
- 截图策略：
  - `fullpage` 优先；
  - 不支持整页时，按顶部/中部/底部分段截图；
  - 禁止仅保留首屏作为页面唯一截图。
- 截图路径：
  - 存在输出文档路径时：`{输出文档目录}/images/`
  - 文档引用相对路径：`images/xx.png`

### 3) 文档生成
- 按文档类型选择结构（快速入门/操作指南/功能说明）。
- 结合 PRD 语义与 UI 实测信息生成步骤：
  - UI 名称、按钮文案、菜单路径必须以实测为准；
  - 步骤包含前提条件、操作动作、预期结果与注意事项；
  - 风险操作明确警示。

## 质量门禁（必须通过）

1. **完整性**：PRD 核心功能均有覆盖章节。
2. **一致性**：步骤与界面文案一致，无杜撰入口或按钮名。
3. **截图可用性**：引用文件存在，关键页面覆盖完整（非首屏单图）。
4. **格式规范**：Markdown 结构正确，标题层级清晰，术语前后一致。

## 异常处理

- PRD 不存在或格式不支持：终止并报错。
- 控制台不可访问：提示用户确认是否仅基于 PRD 继续。
- 页面长期不稳定/入口缺失：记录“已知限制”，继续处理其它模块。
- 需要登录：引导使用已登录浏览器或先完成登录。

## 产物

- 主文档：用户指定输出路径（通常 `raw/.../*.md`）
- 截图目录：主文档同级 `images/`
- 可选元数据：记录 `prd_source/console_url/screenshot_count/sections`

## 协作关系

- 浏览器自动化：`browser-use`
- 可选复检：`doc-consistency-verifier`
